# 🤝 Contributing to Job Rush

First off, thank you for considering contributing to Job Rush! It's people like you that make Job Rush such a great tool for everyone.

## 🌈 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail, including steps to reproduce.
- Mention your OS and Browser version.

### Suggesting Enhancements
- Open a new issue with a clear title and description.
- Explain why this enhancement would be useful.

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies** in both `frontend` and `backend`.
3. **Write clean code**: Follow the existing coding style and ensure no linting errors.
4. **Update documentation**: If you're adding a feature, update the `README.md` or `ARCHITECTURE.md`.
5. **Submit the PR**: Provide a clear description of the changes and link any relevant issues.

---

## 💻 Style Guide

### JavaScript / React
- Use functional components and hooks.
- Use descriptive variable and function names.
- Keep components small and focused.

### CSS / Tailwind
- Use Tailwind utility classes where possible.
- For complex animations, use Framer Motion.

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Reference issues and pull requests liberally after the first line.

---

## 📜 Code of Conduct
Please be respectful and professional in all interactions within the project.
